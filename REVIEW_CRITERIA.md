# リファクタリング・コードレビュー基準

## 1. セキュリティレビュー
- [ ] 入力検証が十分か
- [ ] SQLインジェクション、XSS、CSRFの脆弱性がないこと
- [ ] 認証・認可ロジックが変更されていないこと
- [ ] 機密情報がログ出力されていないこと
- [ ] 依存関係の脆弱性チェック

## 2. パフォーマンスレビュー
- [ ] データベースクエリの最適化（N+1問題なし）
- [ ] インデックスが適切に設定されていること
- [ ] キャッシュ戦略が維持・改善されていること
- [ ] 不要なループや再帰がないこと
- [ ] メモリリークがないこと

## 3. コード品質レビュー
- [ ] コーディング規約に準拠
- [ ] 適切な命名（変数、関数、クラス）
- [ ] 関数が単一責任の原則に従っている
- [ ] 重複コードの排除
- [ ] 適切なエラーハンドリング
- [ ] コメントが必要かつ適切

## 4. エッジケース・境界値レビュー
- [ ] null/undefinedの処理
- [ ] 空配列、空文字列の処理
- [ ] 数値の境界値（0、負数、最大値）
- [ ] 並行処理・ Race Condition
- [ ] タイムアウト・エラーハンドリング

## 5. テストカバレッジ
- [ ] 既存のテストがパスすること
- [ ] 新しいテストの追加
- [ ] エッジケースのテスト
- [ ] エラーパスのテスト

## 6. 互換性レビュー
- [ ] 後方互換性の維持
- [ ] API変更の影響範囲
- [ ] データベーススキーマ変更の影響

## 7. ドキュメンテーション
- [ ] READMEの更新
- [ ] APIドキュメントの更新
- [ ] 複雑なロジックのコメント
